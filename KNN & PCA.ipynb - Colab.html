1 What is K-Nearest Neighbors (KNN) and how does it work?
Definition: KNN is a non‑parametric, instance‑based supervised learning 
algorithm used for classification and regression. It doesn’t make 
explicit assumptions about the form of the decision boundary. 
Working:
You have a training dataset of instances with features and labels (or 
values).
To predict for a new (query) point, compute distances between that 
point and all training samples using some distance metric (e.g., 
Euclidean, Manhattan). 
Select the top K closest (nearest) training points (“neighbors”).
If classification: do majority vote among the neighbors’ labels; if 
regression: take the average (or sometimes weighted average) of the 
neighbors’ target values. 
2) Difference between KNN Classification and KNN Regression
Aspect  KNN Classification  KNN Regression
Output type Discrete class label (category) Continuous value (numeric)
Decision rule   Majority voting among neighbors Averaging (or weighted 
average) of neighbors’ values 
Sensitive to outliers   Misclassified neighbors can sway vote if K 
small    Extreme values among neighbors can pull average, so also 
sensitive if not weighted or many outliers
3) Role of the distance metric in KNN
Ans:-Distance metric defines what “nearest” means. It determines which 
points are considered neighbors. Common choices:
Euclidean distance (straight‑line) 
Manhattan (sum of absolute differences) 
Minkowski (a generalization) 
If features are on different scales, the larger‐scale ones dominate 
distances, which can degrade performance unless features are 
normalized / standardized.
Choice of metric affects sensitivity to outliers, the shape of 
neighborhoods, and ultimately bias vs variance trade‑off.
4) Curse of Dimensionality in KNN
Asn:-As number of features (dimensions) increases:
Distances between all pairs of points tend to become similar (contrast 
between “nearest” and “farthest” distances shrinks), making neighbor 
distinctions less meaningful.
More data is needed to densely populate the space; with limited data, 
neighborhoods become sparse.
Computational cost increases.
This leads to poorer performance, overfitting, and slower inference. 
KNN suffers more in high dimensions. 
5) How can we choose the best value of K in KNN
Ans:-Use cross‑validation on training data: try several values of K (e.
g. 1,3,5,7,…), compute performance (accuracy for classification, MSE 
for regression) and pick the one with best average score. 
Rule of thumb: sometimes K ≈ √n where n is number of training samples. 
Use odd values for K (for binary classification) to avoid ties in 
majority voting.
Consider trade‑off: small K → low bias but high variance (sensitive to 
noise), large K → smoother decision boundary, possibly underfitting.
6) What are KD‑Tree and Ball‑Tree in KNN
Ans:-These are data structures to speed up nearest neighbor search (so 
you don’t have to compute distance to every training point for every 
query).
KD‑Tree (k‑dimensional tree): recursively partition the space by 
axis‑aligned splits. Every node divides space along one feature 
dimension. Enables faster “pruning” of regions that are far from query 
point. 
Ball‑Tree: partitions space using hyperspheres (“balls”) rather than 
axis‑aligned hyperplanes. Each node covers a subset of points within a 
“ball” (center + radius). Subdivision is based on distance / structure 
of data. 
7)When should you use KD‑Tree vs Ball‑Tree
Ans:-KD‑Tree works well when feature dimension is relatively low (say 
up to ~20 or so), and data is fairly evenly distributed, since 
axis‑aligned splits are effective. 
Ball‑Tree tends to perform better in higher dimensions or when the data 
manifold is irregular / not aligned with coordinate axes (so axis 
splits become inefficient). Also useful if using non‑Euclidean metrics.
 8) What are the disadvantages of KNN?
Ans:-Disadvantages of KNN
Slow at prediction time: need to compute distance to many points 
(unless using efficient data structures). High memory usage (store 
entire training set). 
Sensitive to irrelevant / redundant features, as they contribute to 
distance but may not help in classification/regression.
Curse of dimensionality (as above).
Sensitive to the choice of K and the distance metric.
Sensitive to outliers, particularly for small K.
Imbalanced classes: majority class may dominate votes in classification.
9) How does feature scaling affect KNN
Ans:-Very important: without scaling, features with large numeric 
ranges dominate distance computations, skewing neighbor selection.
Common scaling methods: standardization (zero mean, unit variance), 
min‑max scaling.
Feature scaling helps ensure each feature contributes appropriately to 
distance. Without it, prediction performance can degrade significantly.
10) What is PCA?
Ans:- PCA is a dimensionality reduction / feature extraction technique. 
It transforms a dataset with possibly correlated features into a new 
set of uncorrelated variables called principal components. These are 
ordered so that the first component explains the maximum possible 
variance, the second explains the next maximum variance, subject to 
orthogonality constraints, etc. 
11) How does PCA work?
Ans:-Standardize the data (e.g. zero mean, unit variance) if features 
are on different scales.
Compute the covariance matrix (or correlation matrix) of the features.
Compute the eigenvalues and eigenvectors of this covariance matrix.
The eigenvectors correspond to the principal components (directions in 
feature space), the eigenvalues tell how much variance in the data is 
captured by each component.
Sort components by descending eigenvalue; select top m components 
(where m < d) to reduce dimensionality.
Project the original data onto this new m-dimensional subspace (using 
the selected eigenvectors) to get reduced features.
12) Geometric intuition behind PCA?
Ans:-Think of data as cloud of points in high‑dimensional space.
PCA finds a new coordinate system (a rotation) such that most of the 
variance (spread) lies along the first axis (principal component), next 
most along the second, etc.
You are fitting a hyperplane of lower dimension that best captures the 
main directions of variation in the data (in least squares sense).

1 What is K-Nearest Neighbors (KNN) and how does it work?

Definition: KNN is a non‑parametric, instance‑based supervised learning algorithm used for classification and regression. It doesn’t make explicit assumptions about the form of the decision boundary.

Working:

You have a training dataset of instances with features and labels (or values).

To predict for a new (query) point, compute distances between that point and all training samples using some distance metric (e.g., Euclidean, Manhattan).

Select the top K closest (nearest) training points (“neighbors”).

If classification: do majority vote among the neighbors’ labels; if regression: take the average (or sometimes weighted average) of the neighbors’ target values.

  1. Difference between KNN Classification and KNN Regression Aspect KNN Classification KNN Regression Output type Discrete class label (category) Continuous value (numeric) Decision rule Majority voting among neighbors Averaging (or weighted average) of neighbors’ values

Sensitive to outliers Misclassified neighbors can sway vote if K small Extreme values among neighbors can pull average, so also sensitive if not weighted or many outliers

  1. Role of the distance metric in KNN

Ans:-Distance metric defines what “nearest” means. It determines which points are considered neighbors. Common choices:

Euclidean distance (straight‑line)

Manhattan (sum of absolute differences)

Minkowski (a generalization)

If features are on different scales, the larger‐scale ones dominate distances, which can degrade performance unless features are normalized / standardized.

Choice of metric affects sensitivity to outliers, the shape of neighborhoods, and ultimately bias vs variance trade‑off.

  1. Curse of Dimensionality in KNN

Asn:-As number of features (dimensions) increases:

Distances between all pairs of points tend to become similar (contrast between “nearest” and “farthest” distances shrinks), making neighbor distinctions less meaningful.

More data is needed to densely populate the space; with limited data, neighborhoods become sparse.

Computational cost increases.

This leads to poorer performance, overfitting, and slower inference. KNN suffers more in high dimensions.

  1. How can we choose the best value of K in KNN

Ans:-Use cross‑validation on training data: try several values of K (e.g. 1,3,5,7,…), compute performance (accuracy for classification, MSE for regression) and pick the one with best average score.

Rule of thumb: sometimes K ≈ √n where n is number of training samples.

Use odd values for K (for binary classification) to avoid ties in majority voting.

Consider trade‑off: small K → low bias but high variance (sensitive to noise), large K → smoother decision boundary, possibly underfitting.

  1. What are KD‑Tree and Ball‑Tree in KNN

Ans:-These are data structures to speed up nearest neighbor search (so you don’t have to compute distance to every training point for every query).

KD‑Tree (k‑dimensional tree): recursively partition the space by axis‑aligned splits. Every node divides space along one feature dimension. Enables faster “pruning” of regions that are far from query point.

Ball‑Tree: partitions space using hyperspheres (“balls”) rather than axis‑aligned hyperplanes. Each node covers a subset of points within a “ball” (center + radius). Subdivision is based on distance / structure of data.

7)When should you use KD‑Tree vs Ball‑Tree

Ans:-KD‑Tree works well when feature dimension is relatively low (say up to ~20 or so), and data is fairly evenly distributed, since axis‑aligned splits are effective.

Ball‑Tree tends to perform better in higher dimensions or when the data manifold is irregular / not aligned with coordinate axes (so axis splits become inefficient). Also useful if using non‑Euclidean metrics.

  1. What are the disadvantages of KNN?

Ans:-Disadvantages of KNN

Slow at prediction time: need to compute distance to many points (unless using efficient data structures). High memory usage (store entire training set).

Sensitive to irrelevant / redundant features, as they contribute to distance but may not help in classification/regression.

Curse of dimensionality (as above).

Sensitive to the choice of K and the distance metric.

Sensitive to outliers, particularly for small K.

Imbalanced classes: majority class may dominate votes in classification.

  1. How does feature scaling affect KNN

Ans:-Very important: without scaling, features with large numeric ranges dominate distance computations, skewing neighbor selection.

Common scaling methods: standardization (zero mean, unit variance), min‑max scaling.

Feature scaling helps ensure each feature contributes appropriately to distance. Without it, prediction performance can degrade significantly.

  1. What is PCA?

Ans:- PCA is a dimensionality reduction / feature extraction technique. It transforms a dataset with possibly correlated features into a new set of uncorrelated variables called principal components. These are ordered so that the first component explains the maximum possible variance, the second explains the next maximum variance, subject to orthogonality constraints, etc.

  1. How does PCA work?

Ans:-Standardize the data (e.g. zero mean, unit variance) if features are on different scales.

Compute the covariance matrix (or correlation matrix) of the features.

Compute the eigenvalues and eigenvectors of this covariance matrix.

The eigenvectors correspond to the principal components (directions in feature space), the eigenvalues tell how much variance in the data is captured by each component.

Sort components by descending eigenvalue; select top m components (where m < d) to reduce dimensionality.

Project the original data onto this new m-dimensional subspace (using the selected eigenvectors) to get reduced features.

  1. Geometric intuition behind PCA?

Ans:-Think of data as cloud of points in high‑dimensional space.

PCA finds a new coordinate system (a rotation) such that most of the variance (spread) lies along the first axis (principal component), next most along the second, etc.

You are fitting a hyperplane of lower dimension that best captures the main directions of variation in the data (in least squares sense).

13)Difference between Feature Selection and Feature Extraction?

Ans:-Feature Selection Feature Extraction

Picks a subset of original features (keeping them as is) Transforms original features into new ones (linear/non‑linear combinations) Retains interpretability (you know exactly which features are used) Often less interpretable (new features are combinations) Doesn’t change dimensionality by creating new features, only reduces by removing features Can reduce dimensionality but uses linear algebra / transformations to project data Examples: filter methods (e.g. correlation, mutual information), wrapper methods, embedded methods Examples: PCA, LDA, autoencoders, kernel PCA 14) What are Eigenvalues and Eigenvectors in PCA?

Ans:- Given a covariance matrix Σ Σ of size 𝑑 × 𝑑 d×d, eigenvectors are directions (in feature space) that are invariant (up to scaling) under transformation by Σ Σ.

Eigenvalues measure how much variance is along each eigenvector. So a higher eigenvalue means that component explains more of the data dispersion.

In PCA, you use eigenvalues to decide which components to keep.

  1. How do you decide the number of components to keep in PCA?

Ans:-One way is by inspecting the explained variance (eigenvalues). Plot cumulative variance explained vs number of components; pick the smallest number of components that cover, say, 90‑95% of variance.

Use a scree plot: plot eigenvalues in descending order; find an “elbow” where additional components begin diminishing returns.

Sometimes cross‑validation (reconstruction error, downstream model performance) is used.

  1. Can PCA be used for classification?

ans:-PCA itself is unsupervised and doesn’t use label information. It reduces features, but doesn’t classify.

However, PCA is often used as a preprocessing step before classification: reduce dimensionality, remove noise, and then feed transformed data into classification models (including KNN).

It doesn’t directly produce a classifier, but assists by making downstream classification more efficient / less overfitting.

  1. Limitations of PCA?

Ans:-PCA assumes linear relationships; it may not capture non‑linear structure.

PCA uses variance as criterion: features that are highly variable but irrelevant for the class may dominate; features important for classification but with low variance may be dropped.

Sensitivity to scaling of features.

The transformed components are less interpretable than original features.

PCA requires continuous numerical features; categorical data need encoding.

18)How do KNN and PCA complement each other?

Ans:-PCA can be used with KNN to reduce dimensionality first (mitigating the curse of dimensionality), retain the main variance, then apply KNN in the reduced space, which often yields better performance and faster predictions.

PCA helps remove noise, redundant features, so the distance metrics in KNN become more meaningful.

  1. How does KNN handle missing values in a dataset?

Ans)KNN itself doesn’t inherently handle missing values. Some common strategies:

Impute missing values before applying KNN (mean/mode, median, or more advanced methods).

Use only those features that are non‑missing for distance calculations (but then need normalization or careful weighting).

KNN imputation: one uses KNN to estimate (impute) missing values by taking average of nearest neighbors’ values for that feature.

But care needed: missingness pattern may bias distance metric.

  1. Key differences between PCA and Linear Discriminant Analysis (LDA)? Ans:- Aspect PCA LDA Supervised / unsupervised Unsupervised (doesn’t use labels) Supervised (uses class labels) Goal Find directions (components) that maximize variance of features (regardless of class) Find directions that maximize class separability (maximize between‑class variance / minimize within‑class variance)

Output dimension max Up to d (number of features) components At most C‑1 where C = number of classes Use cases Good for general dimensionality reduction, denoising, compression Good when you want to do classification and want features that discriminate classes Sensitivity to labels Not sensitive; labels don’t affect principal components Sensitive; performance depends on how well class distributions differ

from sklearn.datasets import load_iris, make_regression

# 21. Train a KNN Classifier on the Iris dataset and print model accuracy

X_train_i, X_test_i, y_train_i, y_test_i = train_test_split(X_iris, y_iris, test_size=0.2, random_state=42)
knn_clf = KNeighborsClassifier(n_neighbors=3)
knn_clf.fit(X_train_i, y_train_i)
y_pred_i = knn_clf.predict(X_test_i)
print("Iris KNN Classifier Accuracy:", accuracy_score(y_test_i, y_pred_i))

# 22. Train a KNN Regressor on a synthetic dataset and evaluate using MSE

# 23. Train a KNN Classifier using different distance metrics (Euclidean vs Manhattan) and compare accuracy

# 45 Train a KNN Classifier and perform feature selection before training?
import numpy as np

from sklearn.neighbors import KNeighborsClassifier
from sklearn.metrics import accuracy_score

# Load the Breast Cancer dataset

X = cancer.data
y = cancer.target

# Split the data

print(f"Accuracy of KNN with feature selection: {accuracy:.4f}")

#46 Train a PCA model and visualize the data reconstruction error after reducing dimensions4
import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import make_classification
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
from sklearn.metrics import mean_squared_error

# Generate synthetic data
X, y = make_classification(n_samples=500, n_features=30, n_informative=15, n_redundant=5, n_classes=2, random_state=42)

# Scale the data
scaler = StandardScaler()

for n in n_components_list:
    pca = PCA(n_components=n)
    X_reduced = pca.fit_transform(X_scaled)
    X_reconstructed = pca.inverse_transform(X_reduced)

    reconstruction_errors.append(mse)
    print(f"Components: {n}, Reconstruction Error: {mse:.4f}")

plt.plot(n_components_list, reconstruction_errors, marker='o', linestyle='--')
plt.title('Data Reconstruction Error vs. Number of Components')
plt.xlabel('Number of Principal Components')
plt.ylabel('Reconstruction Error (MSE)')
plt.grid(True)
plt.show()

# 47 Train a KNN Classifier and visualize the decision boundary
import numpy as np
import matplotlib.pyplot as plt
from matplotlib.colors import ListedColormap
from sklearn.datasets import load_iris
from sklearn.neighbors import KNeighborsClassifier

# Load Iris dataset (using only first two features for visualization)
iris = load_iris()
X = iris.data[:, :2]
y = iris.target

# Create color maps
cmap_light = ListedColormap(['#FFAAAA', '#AAFFAA', '#AAAAFF'])
cmap_bold = ListedColormap(['#FF0000', '#00FF00', '#0000FF'])

# Create a KNN Classifier
knn_classifier = KNeighborsClassifier(n_neighbors=5)
knn_classifier.fit(X, y)

# Plot the decision boundary
x_min, x_max = X[:, 0].min() - 1, X[:, 0].max() + 1
y_min, y_max = X[:, 1].min() - 1, X[:, 1].max() + 1
xx, yy = np.meshgrid(np.arange(x_min, x_max, 0.02),
                     np.arange(y_min, y_max, 0.02))
Z = knn_classifier.predict(np.c_[xx.ravel(), yy.ravel()])
Z = Z.reshape(xx.shape)

plt.figure(figsize=(10, 8))
plt.pcolormesh(xx, yy, Z, cmap=cmap_light, shading='auto')
plt.scatter(X[:, 0], X[:, 1], c=y, cmap=cmap_bold, edgecolor='k', s=20)
plt.xlim(xx.min(), xx.max())
plt.ylim(yy.min(), yy.max())
plt.title("KNN Classifier Decision Boundary (K = 5)")
plt.xlabel(iris.feature_names[0])
plt.ylabel(iris.feature_names[1])
plt.show()
